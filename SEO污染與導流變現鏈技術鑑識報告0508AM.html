六、鑑識結論

  1. 三個不同入口網域均導向相同 routing ID，非獨立網站行為。
  2. Step2 導流節點完全一致，顯示後端轉發系統集中。
  3. 偽 MEGA 頁面實際為 HTML，非檔案下載，具品牌偽裝與下載誘導目的。
  4. 短網址失效與 copy-paste 指示顯示其具一次性、動態與反追蹤設計。
  5. 密碼壓縮檔與第三方下載域顯示其已具 payload 交付能力。
最終判斷：本案已由技術證據支持「多網域污染 → 單一導流系統 → 統一變現 / payload 交付鏈」成立。

七、建議後續蒐證

  • 擴大測試更多入口網域，統計 routing ID 收斂比例。
  • 保存每層 HTTP header、Location、Content-Type 與截圖。
  • 安全下載 payload 後僅做 hash 與靜態分析，不執行。
  • 記錄 shorturl 失效時間，作為一次性導流證據。
  • 整理成 Word / PDF / 簡報版送交主管機關。
